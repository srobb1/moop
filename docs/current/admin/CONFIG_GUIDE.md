# Configuration Management Guide for Admins

## How Configuration Works

MOOP uses a two-layer configuration system:

1. **`config/site_config.php`** — Default values (do NOT edit unless deploying to a new server)
2. **`config/config_editable.json`** — Your overrides (managed via Admin Dashboard)

On every page load, ConfigManager merges these: editable values override defaults.
Changes to `config_editable.json` take effect immediately — no server restart needed.

## Quick Start

### I want to...

#### Change the site title or admin email
1. Log in as admin
2. Go to **Admin** → **Manage Site Configuration**
3. Edit the field and save

That's it. Changes are saved to `config/config_editable.json` and apply immediately.

#### Add a new sequence type
1. Go to **Admin** → **Manage Site Configuration**
2. Find the **Sequence Types** section
3. Add your new type with pattern, label, and color

#### Add a new tool
1. Create your tool file: `tools/my_tool.php`
2. Edit `config/tools_config.php` and add an entry:
```php
'my_tool' => [
    'id'              => 'my_tool',
    'name'            => 'My Tool',
    'icon'            => 'fa-star',
    'description'     => 'What it does',
    'btn_class'       => 'btn-primary',
    'url_path'        => '/tools/my_tool.php',
    'context_params'  => ['organism'],
    'pages'           => 'all',
],
```
3. Tool now appears in the UI on the next page load.

#### Disable a tool temporarily
Edit `config/tools_config.php` and comment out the tool entry:
```php
// 'blast_search' => [  // DISABLED - tool won't show
//     ...
// ],
```

#### Deploy to a different site directory
If you want to run this app as a different site (not "moop"):

1. Copy the moop directory: `cp -r /var/www/html/moop /var/www/html/my_site`
2. Edit `config/site_config.php` in the new copy
3. Change: `$site = 'my_site';` (was `'moop'`)
4. All paths and URLs automatically update

This is the ONE case where you edit `site_config.php` — it's a one-time structural
setting at deployment time.

## File Reference

### `config/site_config.php`
Default application configuration. Contains paths, URLs, and default settings.

**Do NOT edit this file** to change settings like site title, email, or sequence
types. Use the Admin Dashboard instead. The only values an admin might change here
are `$root_path` and `$site` when deploying to a new server.

### `config/config_editable.json`
Admin-editable overrides. Managed via Admin Dashboard → Manage Site Configuration.

**Editable settings:**
- `siteTitle` — site name shown in browser tab and navigation
- `admin_email` — contact email
- `sequence_types` — file types and their display labels/colors
- `header_img` — header image filename
- `favicon_filename` — favicon filename
- `auto_login_ip_ranges` — IP ranges for automatic login
- `sample_feature_ids` — sample IDs for sequence retrieval
- `blast_sample_sequences` — sample sequences for BLAST

### `config/tools_config.php`
Tool registry. Defines which tools are available and where they appear.

**What to edit:**
- Add new tool entries following the template
- Change `name`, `icon`, `description`, `btn_class` to customize appearance
- Change `pages` to control where each tool shows
- Comment out tools to disable them

### `includes/ConfigManager.php`
Core configuration manager class. Loads and merges config layers. Used by developers,
not directly by admins.

### `includes/config_init.php`
Initialization file. Loaded once per page automatically. Read-only.

## Accessing Config in Code

### Get a config value
```php
$config = ConfigManager::getInstance();
$title  = $config->getString('siteTitle');
$path   = $config->getPath('organism_data');
$arr    = $config->getArray('sequence_types');
$bool   = $config->getBoolean('jbrowse2.enabled');  // dot-notation for nested
```

### Check which pages a tool appears on
Tools config uses the `pages` setting:
- `'pages' => 'all'` — shows on every page
- `'pages' => ['index', 'organism']` — shows only on those pages
- Omitting `pages` defaults to `'all'`

Available page names: `index`, `organism`, `group`, `assembly`, `parent`, `multi_organism_search`

## Security

### Access Control is Separate
- User permissions and access levels are in `includes/access_control.php`
- Assembly-level access is controlled by `metadata/organism_assembly_groups.json`
- **The configuration system does NOT touch user access**

### What ConfigManager handles:
- Site paths and URLs
- Application settings (title, email, etc.)
- Tool registry (which tools exist)
- Data configurations (sequence types)

### What ConfigManager does NOT handle:
- User authentication
- User permissions (PUBLIC / COLLABORATOR / ADMIN / IP_IN_RANGE)
- Assembly-level access control
- Session data

## Common Issues & Solutions

| Problem | Solution |
|---------|----------|
| Tool doesn't appear | Check `pages` setting in tools_config.php. User may lack permission. |
| Site won't load | Check paths in site_config.php exist on server. Check web server error logs. |
| Config changes don't apply | Clear browser cache. Check config_editable.json is valid JSON. |
| Not sure what a setting does | See comments in site_config.php — each setting is documented inline. |
| Can't add new tool | Ensure entry in tools_config.php matches template exactly. Check tool file exists. |
| Site title won't change | Use Admin Dashboard → Manage Site Configuration (not site_config.php). |
| Paths are wrong | Check `$root_path` and `$site` at top of site_config.php. All other paths auto-calculate. |

## How a New Admin Gets Started

1. **First Day:** Read this guide (5 min)
2. **Change settings:** Admin Dashboard → Manage Site Configuration
3. **Add a tool:** Follow "Add a new tool" section above
4. **Something broken:** Check "Common Issues" table above
5. **Full details:** Read the inline comments in `config/site_config.php`
